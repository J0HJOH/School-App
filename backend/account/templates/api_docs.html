GoDocs API Documentation

Introduction

GoDocs is an efficient API tool that provides a user friendly way of building school apps. it is powerful and highly flexible and at the same time ensuring its credibilty in securing users.

Reasons why you may want to use GoDocs
  • Extensive documentation and great community support.
  • Easy to use and customizable all the way down.
  • Authentication policies are great, providing ways of generating API keys and tokens securely and also in a programatic way.

Get Started

On Signing Up for an account on GoDocs you will be redirected to your dashboard where you will be able to generate an API key or a token. Ensure you keep your tokens and keys save for future refrence and for security purposes. Follow the required instructions on how to generate a key savely.

How to create users using the API Endpoint

Make a Post request using the endpoint:

{{request.META.HTTP_HOST}}{% url 'api:create' %}

Ensure you read the On how to make an Authorized request with your API keys before procceding with your post request.

The underlisted parameters are needed when creating a user:

  • The User's Name
  • The User's Email Address
  • The 'User's Department
  • The 'User's Password

The User's Name and Password is made to be unique accross the board. That means that, not more than one account can be created with the same name or email. This is to ensure that, no user can have more one account. If for instance a user input the same name or email or both name and email that has been created and saved before in the database, a status code of 400 is retured showing a bad request and returing a response showing the following:

{{ "name": [ "Name must be unique" ],
"email": [ "An account has already been created with this email" ] }}

More importantly, GoDocs added a layer of security to the user's Password ensuring that the User inputed a strong password. Therefore, if a request is been made with a poor password, a status code of 400 will be shown. This implies that, a poor request has been made and in return it shows an error message.

How to get list of all the users

In a similar way to how we created users using the api endpoint, we can also get a list of all registered users. But this time, we are going to Make a get request to the endpoint below:

{{request.META.HTTP_HOST}}{% url 'api:list' %}

To Update Users Credentials

Similar to how we created users you can also update a user credentials, simply by making a 'PUT' request to the endpoint:

{{request.META.HTTP_HOST}}{% url 'api:update' id=request.user.id %}

Ensure you replace '{{request.user.id}}' with the user's id and add the field and also the value of the field that you want to update to the body similar to how it was done when we created users but this time selectively pick the field that you want to update. For instance; if you only want to update the name of the user, only add name as the key to the body and let the name of the user be the value. In a similar way, you can update several fieldset at the same time by inputing all the fieldset of the user that you want to update and the value to the body.

Note: More importantly, the password fieldset can not be updated using this route. It can only be updated if the user request for a password reset incases were the user forgets their password. A verfication email and a link will be sent to the user via email were the user will be able to change his/her password. Read on the various ways you can choose for your users to reset their password.

To Retrive A Particular User

Similar to how we performed a 'GET' request to get theb list of all users you can also get a user's identity by making a get request to the endpoint below:

{{request.META.HTTP_HOST}}{% url 'api:retrive' id=request.user.id %}

Replace '{{request.user.id}}' with the user's id
